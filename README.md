# Дайджест ИТ-новостей (three-source-digest)

Лёгкая версия бота: собирает новости из **трёх источников** и присылает в тот же
Telegram-канал через того же бота.

Источники:
- CNews — `https://www.cnews.ru/inc/rss/news.xml`
- TAdviser — `https://www.tadviser.ru/xml/tadviser.xml`
- Новости ИТ-канала — `https://www.novostiitkanala.ru/rss`

Формат каждой новости:
```
🔹 Заголовок новости
Одна фраза сути (из описания RSS; если его нет — только заголовок)
Источник →   (ссылка)
```

В канал попадают **только новые** новости: бот помнит уже отправленное в файле
`seen.json`, поэтому одни и те же новости не приходят повторно.

## Как это устроено

- `news_bot.py` — весь код в одном файле, из внешних сервисов только Telegram.
- Загрузка лент: `requests`, а при сбое TLS — фолбэк на системный `curl`
  (нужно на старом macOS-Python; в GitHub Actions работает напрямую).
- «Одна фраза» берётся из описания RSS. Мусорные вики-ссылки TAdviser
  («Основная статья: …») и описания, дублирующие заголовок, отбрасываются.
- `seen.json` — список id уже отправленных новостей (хранится последних ~1500).
- Новости старше 3 дней игнорируются (защита от потопа).

### Где добавлять фильтр по темам

Когда понадобится отсекать часть новостей — вся логика живёт в одной функции
`should_include(item)` в `news_bot.py`. Сейчас она пропускает всё.

## Запуск локально

```bash
pip install -r requirements.txt

# положить токен и chat_id в .env:
#   TELEGRAM_BOT_TOKEN=...
#   TELEGRAM_CHAT_ID=...

# посмотреть дайджест без отправки:
python3 news_bot.py --dry-run

# отправить в Telegram:
python3 news_bot.py
```

### Первый запуск: «засеять» память

Чтобы при старте не пришёл разом весь список за 3 дня, один раз пометьте всё
текущее как уже отправленное:

```bash
python3 news_bot.py --seed
```

После этого бот будет слать только то, что появится дальше.

## Автозапуск (GitHub Actions)

Расписание — дважды в день: **08:00 и 20:00 МСК** (`.github/workflows/daily-news.yml`).
Всё крутится на серверах GitHub — компьютер может быть выключен.

Чтобы заработало:
1. Смёржить ветку в `main` (планировщик Actions запускается только с ветки по умолчанию).
2. В настройках репозитория задать Secrets: `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID`.
3. Включить Actions.

Память об отправленном (`seen.json`) экшен коммитит обратно в репозиторий — так
состояние сохраняется между запусками, а заодно репозиторий остаётся активным
(GitHub не отключит расписание по «правилу 60 дней тишины»). Право записи для
экшена уже выставлено (`default_workflow_permissions: write`).

Ручной запуск — кнопка **Run workflow** во вкладке Actions.
